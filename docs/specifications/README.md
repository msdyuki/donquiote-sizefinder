# プロジェクト仕様書

ここは「ドン・キホーテ（PPIH様）デジタル採寸サイネージ」プロジェクトの仕様を格納・管理するためのディレクトリです。

## 今後格納されるドキュメントの例

- **要件定義書**: システムの目的や必須機能、非機能要件など
- **画面設計書 (UI/UX)**: 画面遷移図や各画面のワイヤーフレーム
- **システムアーキテクチャ設計**: 使用する技術スタック、フロントエンド/バックエンド構成
- **API仕様書**: 外部システム（採寸データAPIなど）との連携仕様
- **デザインガイドライン**: テーマカラー、フォント、アニメーションのルール

※ 新しい仕様書を作成する際は、このディレクトリ (`docs/specifications/`) 内に Markdown ファイル等で追加してください。

## 開発・動作確認手順

本プロジェクトでは、サイネージ本体とスマホ用LP（ランディングページ）の2つのビルドが存在します。それぞれのローカルでの動作確認方法は以下の通りです。

### 1. サイネージ（本体）の確認
- **開発モード**: `npm run dev` を実行し、ブラウザで `http://localhost:5173/` にアクセスします。
- **ビルド**: `npm run build:signage`

### 2. スマホ用LP（ランディングページ）の確認
LPはJavaScriptが外部モジュールとして出力されるため、ダブルクリック（`file://`プロトコル）ではCORSエラーとなり表示されません。**必ずローカルサーバー経由**で確認してください。

- **確認手順**:
  1. `npm run build:lp` を実行し、`dist-lp` フォルダに成果物を生成します。
  2. `npx serve dist-lp -p 3000` を実行してローカルサーバーを起動します。
  3. ブラウザで `http://localhost:3000/lp.html` にアクセスします。
- **デプロイ先**: `https://pdc-dev.sakura.ne.jp/ppih/lp.html` 等のWebサーバー（※実運用に合わせて適宜更新）
